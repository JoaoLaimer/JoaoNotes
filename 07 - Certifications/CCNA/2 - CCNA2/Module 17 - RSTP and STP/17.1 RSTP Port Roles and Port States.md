RSTP (IEEE 802.1w) supersedes the original 802.1D standard but retained the following to maintain backward compatibility.
However, RSTP can achieve much faster convergence in a properly configured network.


| STP specific                            | STP 802.1D                                                                      | RSTP 802.1w                                                                                           |
| --------------------------------------- | ------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| BPDUs Forwarding (In a stable topology) | Only the root bridge sends BPDU.<br>Root BPDUs are relayed by other switches.   | All switches send Hello BPDUs every 2 seconds.<br>Used as a "keepalive" mechanism.                    |
| Port States                             | - Disabled.<br>- Blocking.<br>- Listening.<br>- Learning.<br>- Forwarding.      | - Disabled.<br>- Discarding.<br>- Forwarding.                                                         |
| Port Roles                              | - Root (Forwarding)<br>- Designated (Forwarding)<br>- Non-Designated (Blocking) | - Root (Forwarding)<br>- Designated (Forwarding)<br>- Alternate (Discarding)<br>- Backup (Discarding) |
| Time to start forwarding frames         | Up to 50 seconds with default timer settings.                                   | Within a few seconds.                                                                                 |
# 17.1.2 RSTP Port States

| Port States | Description                                                                                                                                                                                                                                                                           |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Disabled    | - Same as the STP Disabled state.<br>- This state is not specific to RSTP but indicates that the port has been administratively disabled by an administrator or due to a hardware failure.<br>- The port does not forward frames.                                                     |
| Discarding  | - Same as the STP Blocking state.<br>- The port is blocked to prevent loops.<br>- The port does not forward frames.<br>- The port listens to BPDUs to learn the STP topology.                                                                                                         |
| Forwarding  | - Similar to the STP Forwarding state but also incorporates the Learning state.<br>- The port immediately starts forwarding frames.<br>- The port also learns the MAC addresses of connected device(s).<br>- The port continues to process BPDUs monitoring for STP topology changes. |
# 17.1.3 RSTP Port Roles

| Port Role       | Description                                                                                                                                                                                                                                                                                                                                               |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Root Port       | - The root port forward traffic to the root bridge.<br>- The root port offers the lowest path cost from the switch to the root bridge.<br>- Each non-root bridge must have one root port.                                                                                                                                                                 |
| Designated Port | - The designated port forwards traffic from a segment to the root bridge.<br>- The designated port is determined through the bridge's Root Path Cost and Bridge Identifier (Bridge ID) compared to other bridges on the segment.<br>- All STP enabled switches have one or more designated ports.<br>**Note**: The root bridge only has designated ports. |
| Alternate Port  | - Alternate ports provide another paths to the root bridge if the root port fails.<br>- Alternate ports are in discarding state and quickly transition to the forwarding state when the root port path fails.<br>- Alternate ports are typically found on non-root bridges and help ensure rapid convergence in the event of a link failure.              |
| Backup Port     | - Backup ports serve as a backup to designated ports on a switch.<br>- Backup ports discard traffic and immediately transition to the forwarding state if the designated port fails.<br>**Note**: Backup ports only apply when a hub is connected to an STP switch.                                                                                       |
# 17.1.4 RSTP Link Types and Edge Ports
To achieve fast convergence, the protocol relies on:
- **Link type**: Link type can be point-to-point or shared as determined by the port duplex mode. Full duplex indicates a P2P link, while a half duplex port indicates a shared port.
- **Edge port**: Access ports connecting to one device (e.g., host, server, ...) are edge ports. All other ports are non-edge ports. Edge ports immediately transition to the forwarding state when enabled (skipping the listening and learning state). However, if an edge port ever receives a BPDU, it immediately loses the edge port status.
