# 15.7.6 OAuth Authentication

OAuth is a secure protocol that allows third-party applications to access a user's resources without exposing their credentials. It uses access tokens issued after an initial authentication process.

**Key Features of OAuth:**

- Enables Single Sign-On (SSO)
- Integrates with social media platforms
- Facilitates communication between services in microservices architectures
- Provides granular access control
- Enhances user experience while maintaining security

**Versions of OAuth:**

- **OAuth 1.0a:** Uses signatures for authentication
- **OAuth 2.0:** More streamlined, uses tokens instead of signatures, and is widely adopted

### **How OAuth 2.0 Works**

OAuth 2.0 lets users access different applications and web services using existing credentials from providers like Google or Facebook. 

1. **Authorization Request:**
    - User clicks "Log in with Google" on the music platform.
    - The music platform redirects the user to Google's login page, requesting access to their Gmail profile and email.
2. **Authorization Grant:**
    - User logs into Gmail and grants permission.
3. **Access Token Request:**
    - The music platform sends the authorization code and its own credentials to Google to request an access token.
4. **Access Token:**
    - Google verifies the code and the music platform's identity.
    - Google issues an access token to the music platform.
5. **Resource Request:**
    - The music platform uses the access token to request the user's profile and email from Google.
6. **Protected Resource:**
    - If the access token is valid, Google provides the requested information to the music platform.
