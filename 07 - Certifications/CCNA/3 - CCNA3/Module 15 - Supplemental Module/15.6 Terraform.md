# 15.6.1 Introduction to Terraform
Terraform is an open-source tool developed by HashiCorp, designed to help professionals efficiently build, change, and version infrastructure safely and predictably.
Terraform, like Ansible, is a tool for Infrastructure as Code (IaC). It automates the setup and management of networking resources, such as routers, switches, and firewalls, in both on-premises and cloud environments. Using Terraform ensures consistent and repeatable network configurations. This aligns with modern DevOps practices, which emphasize automation and monitoring in software and infrastructure management.

# 15.6.2 Terraform Architecture

Ansible uses SSH to directly access devices, making it ideal for configuration management on systems where SSH is the main interface, like traditional network devices and servers. It executes commands and scripts directly on the target device, allowing for straightforward automation of configuration changes. In contrast, Terraform uses providers, which are plugins that interact with devices through APIs like RESTCONF or NETCONF.

A Terraform provider is essentially a plugin that allows Terraform to interact with different technologies. For example, Cisco offers several Terraform providers to configure devices.

Like a plugin, a Terraform provider is software that adds specific features to an existing computer program, enhancing its capabilities without modifying the core application. The Terraform provider enables Terraform to communicate with and manage technologies through their specific APIs.

The Terraform provider translates Terraform configurations written in HashCorp Configuration Language (HCL) into commands that specific technologies, like a Cisco router or a cloud service, can understand and execute. For example, Cisco offers Terraform providers to use with Cisco Intersight Infrastructure Service and Cisco Meraki. These providers use APIs to manage tasks like setting up network configurations and managing cloud infrastructure.

# 15.6.3 HashiCorp Configuration Language (HCL)

HCL is a configuration language designed to be both human-readable and machine-friendly, similar to JSON and XML. This makes it ideal for writing configuration files to manage infrastructure. As you explore network automation and IaC, understanding HCL, YAML, and other data structures will help you use tools like Terraform and Ansible effectively.

HCL is similar to JSON in data structure and uses braces for blocks and lists. However, HCL also includes features for configuration management, such as comments and a more flexible syntax.
```
# Assuming there is a provider that supports Cisco IOS XE RESTCONF
provider "cisco_ios_xe" {
  host     = "192.168.10.1"  
  username = "admin"  
  password = "adminpass"  
  secure   = true  # Assuming secure implies HTTPS/RESTCONF is being used securely
}
# Resource configuration for an interface
resource "cisco_ios_xe_interface" "gigabit1" {
  name        = "GigabitEthernet1"  
  description = "Uplink Interface"  
  enabled     = true
} # Output the operational status of the interface
output "interface_status" {
  value = cisco_ios_xe_interface.gigabit1.operational_status  
  description = "Operational status of the GigabitEthernet1 interface."
}
```

# 15.6.5 Network Management Tools and Protocols
**RESTCONF and NETCONF**

RESTCONF and NETCONF are network configuration protocols that allow direct interaction with network devices. They provide standardized APIs for secure communication, used to modify device configurations and retrieve operational data.

**Ansible**

Ansible is an automation platform that uses YAML for its playbook scripts. It excels in configuration management, application deployment, and orchestrating IT workflows. Ansible can interact with network devices using SSH or RESTCONF/NETCONF, making it effective for network automation tasks.

**Python**

Python is a programming language widely used for scripting automation tasks. It supports various libraries like netmiko, NAPALM, NORNIR, and ncclient, which help automate network operations, including those involving RESTCONF and NETCONF.

**Terraform**

Terraform is an infrastructure as code tool that manages the lifecycle of IT and network resources through declarative configuration files written in HCL. It can manage API-driven network components like firewalls, load balancers, and some routers and switches. Terraform provisions underlying infrastructure (e.g., VMs, networks, cloud resources), while tools like Ansible handle detailed configurations.
