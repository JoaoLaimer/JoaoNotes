# 9.1.1 Default Gateway Limitations
If a router or a router interface (that serves as a default gateway fails), the hosts configured with that default gateway are isolated from outside networks. The First Hop Redundancy Protocol (FHRPs) is needed to provide alternate default gateways in switched networks where two or more routers are connected to the same VLANs.
# 9.1.2 Router Redundancy
One way to prevent a single point of failure at the default gateway is to implement a virtual router. To implement this type of router redundancy, multiple routers are configured to work together to present the illusion of a single router. By sharing an IP address and MAC address, two or more routers can act as a single virtual router.

A redundancy protocol provides the mechanism for determining which router should take the active role in forwarding traffic. It also determines when the forwarding role must be taken over by a standby router.

# 9.1.3 Steps for Router Failover
1. The standby router stops seeing Hello messages from the forwarding router.
2. The standby router assumes the role of the forwarding router.
3. Because the new forwarding router assumes both the IPv4 and MAC addresses of the virtual router, the host devices see no disruption in service.
# 9.1.4 FHRP Options

| FHRP Options                                          | Description                                                                                                                                                                                                                                          |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hot Standby Router Protocol (HSPR)                    | HSPR is a Cisco-proprietary FHRP that is designed to allow for transparent failover of a first-hop IPv4 device. HSRP is used in a group of routers for selecting an active device and a standby device.                                              |
| HSRP for IPv6                                         | An HSRP IPv6 group has a virtual MAC address derived from the HSRP group number and a virtual IPv6 link-local address derived from the MAC address. Periodic RA are sent for the HSRP virtual IPv6 link-local address when the HSRP group is active. |
| Virtual Router Redundancy Protocol Version 2 (VRRPv2) | This allows several routers on multiaccess link to use the same virtual IPv4 LAN. A VRRP router is configured to run in conjunction with one or other routers attached to a LAN. Masters and backups.                                                |
| VRRPv3                                                | Supports IPv6 and works in multi-vendor environmnets.                                                                                                                                                                                                |
| Gateway Load Balancing Protocol (GLBP)                | FHRP that protects data traffic from a failed router or circuit, while also allowing load balancing.                                                                                                                                                 |
| GLBP for IPv6                                         | Supports IPv6.                                                                                                                                                                                                                                       |
| ICMP Router Discovery Protocol (IRDP)                 | RFC 1256. Legacy FHRP solution.
