Process that allows for the linking of electronic identities and attributes to store information across multiple distinct identity management systems.
Federation works by using the trust relationships that exist between different systems.
It acknowledges the need for networks to be open to a wider audience, including partners, suppliers, and customers.
A supplier or customer can authenticate using their own network credentials.
1. **Login initiation**
	The user accesses a service or application and chooses to log in.
2. **Redirection to an IdP**
	The service provider redirects the use to the IdP for authentication.
3. **Authenticating the user**
	After a user submits credentials to the IdP, it validates the user's identity.
4. **Generation of an assertion**
	 The IdP creates an assertion that includes information about the user's identity.
5. **Returning to a service provider**
	The user is redirected back to the service provider with the authentication assertion form the IdP.
6. **Verification and access**
	The service provider checks the assertion from a trusted IdP and grants access based on its information.
