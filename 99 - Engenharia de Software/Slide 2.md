### Ciclo de Vida do Software
O software tem um ciclo de vida: 
- é concebido;
- é desenvolvido;
- entra em operação;
- é retirado de operação;

TRÊS ETAPAS: definição, desenvolvimento e
manutenção
## Definição (Concepção)
Estudo de viabilidade: econômica, legal, tecnológica
### Análise de requisitos
Estabelecer quais as funcionalidades requeridas e as restrições sobre a operação do sistema e seu desenvolvimento.

**Requisito** = condição necessária para obtenção de certo objetivo.

- São objetivos ou restrições estabelecidas por clientes e usuários do sistema que definem as diversas propriedades do sistema (e que dizem respeito ao software).
-  Uma condição ou uma funcionalidade necessária a um usuário para resolver um problema. 
Tanto o cliente como o analista desempenham um papel ativo na análise e especificação de requisitos.

**FASE IMPORTANTE, ERROS AQUI, PODEM TRAZER PROBLEMAS NAS OUTRAS FASES DO DESENVOLVIMENTO**

### Descoberta de requisitos
Obter requisitos do sistema, pela observação, conversa com usuários, pela análise de tarefas.

### Especificação de requisitos
Representar os requisitos, em texto ou notações gráficas.

## Desenvolvimento
Implementação e teste, integração e implantação.
- Implementação: Traduz a estrutura definida durante o projeto em um programa executável. Envolve a programação, depuração e testes.
- Integração + instalação: integração dos módulos, testes, implantação e instalação do sistema.
### Teste de Software
Verificação e validação para mostrar que o sistema está de acordo com a especificação.

### Integração
Processo de "juntar" todas as partes do sistema. Realizado de maneira incremental.

### Instalação
Colocar me operação, onde aparecem os erros do analista.

### Manutenção
Após a entrega do sistema e o início da operação do mesmo.

### Evolução do Sistema
Problema: evolução é custosa:
-  Mudanças devem ser analisadas técnica e economicamente.
- Interação entre novos sub-sistemas podem causar novos problemas.
- A estrutura do sistema se corrompe a medida que se realiza mudanças.
## Esforço de produção
**Definição e Desenvolvimento (30% do esforço)**:
- inicio: quando produto é requisitado
- fim: quando produto é entregue para operação
**Operação/Manutenção** (70% do esforço):
- Aumento da capacidade do produto (60%)
- Adaptação do produto a novos ambientes (20%)
- Correção de erros (20%)

## Resumo
Definição -> Analise de requisitos -> Desenvolvimento -> Implementação -> Instalação -> Operação -> Manutenção
