## Engenharia de Software
Se dedica a projetar, implementar e modificar software, de forma que ele seja de alta qualidade, a um custo razoável, manutenível e rápido de construir (Laplante, 2007),

É a aplicação de abordagens sistemáticas, disciplinadas e quantificáveis (...) ao desenvolvimento de software (...) (IEEE ComputerSociety, 2004).

### Atividades
- Analisar
- especificar
- projetar
- programar (implementar)
- verificar
- validar
- manter e gerenciar projetos de software

### Papéis na Equipe
- Gerente de Projeto: garante cumprimento de prazos e orçamentos.
- Analista: levantamento de requisitos e sua devida modelagem -> especificação do software
- Projetista: com base na especificação do analista, propor soluções (tecnologia, arquitetura do sw, etc).
- Desenvolvedor: constrói a solução física a partir da especificação construída pelo projetista. 
- Testador: plano dos testes, execução de testes
- Engenheiro de Software
- Designer

## Tipos de Software
- Genéricos
	Software para linha de produto, funcionalidade especificar que atendam a necessidade de MUITOS clientes. Ex: anti-virus, planilha eletrônica, editor de texto, etc.
- Customizados (encomendados)
	Software de aplicação, sistemas customizados para necessidade de UM cliente, desenvolvido sob demanda.
## Categorias de Software
- **Software básico ou de sistema**: compiladores, driver, componentes do S.O como gerenciadores de arquivos
- **Software de aplicação:** soluções para uma aplicação específica de negócio (controle de estoques, vendas, etc)
- **Software científico/ de engenharia**: processamento de números, projeto apoiado por computador, simuladores, etc.
- **Software embarcado**: software presentes em maquina de lavar, automóveis e etc.
- **Software para linha de produtos**: componentes reutilizáveis, atender múltiplos clientes.
- **Aplicações web/Móveis**: sw voltado as redes, navegadores, computação na nuvem,etc.
- **Software de inteligência artifical**: uso de heurísticas

## Princípios da Eng. de SW
**Decomposição**: software é um produto complexo que de ser construído de partes menores
**Abstração**: auxilia na compreensão de estruturas grandes e complexas.
**Generalização**; produzir soluções genéricas (re-uso).
**Flexibilização**: processo e produto devem permitir atualização e incorporação de modificações.
**Formalidade**: utilização de métodos "formais".
**Desenvolvimento iterativo**: em vários ciclos. Cada ciclo contribui.
